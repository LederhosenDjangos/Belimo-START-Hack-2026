# Belimo START Hack 2026

Leakage detection demo for Belimo actuator data using real-time telemetry and a simulated differential-pressure scenario.

## Project Goal

This project demonstrates a practical leakage response workflow for HVAC/data-center pipelines:

- Monitor actuator telemetry in real time.
- Simulate differential pressure (`∆p`) behavior.
- Compute pressure derivative (`d∆p/dt`).
- Trigger emergency valve closing when `d∆p/dt` leaves configured safety bounds.

## Business Case

Assumption: a sudden pressure drop indicates leakage.

Response strategy:

1. Track `d∆p/dt` against upper/lower bounds.
2. If derivative is outside bounds, immediately force valve setpoint to `0%`.
3. Keep visual evidence and export data for reporting.

## Main Simulation (`sim.html`)

The main simulation in this repository is [`sim.html`](sim.html).

It is a standalone browser-based simulation for the rack-cooling leakage use case and is the best entry point for understanding the concept quickly.

- Interactive scenario switching for normal operation, tray installation, and leakage / pipe rupture
- Real-time plots for supply and return pressure (`p1`, `p2`)
- `dp/dt` visualization for leakage detection
- Annotated Belimo Energy Valve and server-rack schematic
- Parameter controls for hydraulic setup, dynamic response, and alarm threshold

## Data Flow

- Commands are written via `set_process_data(...)` to InfluxDB (`_process`).
- Telemetry is read via `get_measurement_data(...)` from InfluxDB (`measurements`).
- `sim.html` provides the main standalone simulation in the browser.
- In the Streamlit app:
  - Real-time plot is fully selectable (`x` and `y`).
  - Leakage example simulates `∆p` over time.
  - `d∆p/dt` is computed from simulated `∆p`.
  - Out-of-bound derivative triggers emergency close.

## Demo Overview

- `sim.html`: main simulation for the hydraulic model and leakage-detection story
- `demo/app.py`: Streamlit demo for live actuator telemetry, setpoint control, and CSV export

## Main Demo Features (`demo/app.py`)

- Header: **Belimo Case**
- Section: **Leakage Detection: Derivative Bounds on Differential Pressure**
- Controls:
  - Normal valve setpoint (%)
  - Lower/upper bound for `d∆p/dt`
  - Start/Stop `d∆p/dt` example
  - Manual emergency close / reset
- Plots:
  - Real-time actuator plot (`x/y` selectable)
  - Actuator window: setpoint vs feedback during simulation
  - `∆p` over elapsed simulation time
  - `d∆p/dt` with bounds
- Export on stop:
  - Saves CSV files into `demo/assets/`

## Generated CSV Exports

When pressing **Stop d∆p/dt Example**, the app overwrites:

- `demo/assets/actuator_position_window.csv`
- `demo/assets/delta_p.csv`
- `demo/assets/delta_p_derivative_bounds.csv`

## Recreate PNG Plots from CSV

Script:

- `demo/assets/generate_plots_from_csv.py`

Run:

```powershell
python demo/assets/generate_plots_from_csv.py
```

This generates:

- `demo/assets/actuator_position_window.png`
- `demo/assets/delta_p.png`
- `demo/assets/delta_p_derivative_bounds.png`

## Run Locally (venv)

From repository root:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install streamlit pandas altair influxdb-client matplotlib
streamlit run demo/app.py
```

To use the main simulation, open `sim.html` directly in a browser.

## Run with Docker

From `demo/`:

```powershell
docker build -t demo .
docker run --rm -it -p 8501:8501 -v ${PWD}:/work demo
```

Then open:

- `http://localhost:8501`

## Notes

- Bounds default to `-0.3` and `+0.3` (`bar/s`).
- The simulated `∆p` sequence includes a stable segment, sudden drop, and extended steady tail.
